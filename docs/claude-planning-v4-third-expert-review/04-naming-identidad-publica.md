# Naming e identidad pública

## Propuesta actual

```text
ARC Flow
ARC = Agentic Release Coordination
skills: /arc-*
launcher: arcflow
```

## Problema

`ArcFlow` ya se utiliza en distintos productos y proyectos relacionados con:

- agentes;
- workflows;
- AI workforce;
- SDKs;
- automatización;
- plataformas operacionales.

Esto genera riesgos:

- colisión del binario;
- confusión en búsquedas;
- disponibilidad limitada de paquetes;
- identidad pública débil;
- dificultad para registrar dominios;
- potenciales riesgos comerciales.

## Alcance de la observación

No se afirma que exista una infracción legal.

Una evaluación legal requeriría:

- búsqueda marcaria;
- territorios;
- clases;
- uso efectivo;
- asesoría especializada.

La observación es técnica y estratégica.

## Naming gate recomendado

Validar:

- GitHub;
- npm;
- PyPI;
- crates.io;
- Homebrew;
- Chocolatey;
- dominios;
- marketplaces;
- motores de búsqueda;
- marcas relevantes;
- binarios instalables.

## Criterios del nombre

El nombre debería ser:

- distintivo;
- corto;
- pronunciable;
- buscable;
- disponible;
- no atado exclusivamente a Claude;
- compatible con plugin y CLI;
- sin duplicación incómoda en slash commands.

## Decisión

No utilizar todavía `ARC Flow` como nombre definitivo.

Mantenerlo como codename hasta cerrar el naming gate.
