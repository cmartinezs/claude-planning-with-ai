name: plan-export
description: Export a planning as a formatted document — PR description, ticket list, or standalone markdown summary.
argument-hint: <NNN-slug> [--format pr|tickets|markdown]
allowed-tools: [Read, Bash, Glob]
---

Export a planning's content as a formatted document for use outside the planning system.

## Arguments

`$ARGUMENTS` — format: `NNN-slug [--format pr|tickets|markdown]`
- `NNN-slug` — planning id (e.g. `001-jwt-auth-api`)
- `--format pr` — GitHub Pull Request description (default if in a git repo with a remote)
- `--format tickets` — numbered ticket list suitable for Linear or Jira
- `--format markdown` — standalone markdown document (default if no format specified)

## Steps

1. Parse `$ARGUMENTS`: extract planning id and optional `--format` flag. Default format: `markdown`.

2. Locate the planning in `.planning/` (INITIAL), `.planning/active/`, or `.planning/finished/`. If not found, stop and report.

3. Read `00-initial.md` (intent, why, scope), `01-expansion.md` (scope list, dependencies), and all `02-deepening/*.md` scope files (status, done criteria, tasks).

4. **Format: `markdown`** — output a standalone document:
```
# <planning-id>: <intent>

**Why:** <why from 00-initial.md>

## Scopes

| Scope | Area | Status | Summary |
|-------|------|--------|---------|
| scope-01-docs | DO | DONE | … |
…

## Done Criteria (per scope)

### scope-01-docs
- [ ] Auth contract documented in docs/api/auth.md
…

## Open Questions
<from 00-initial.md>
```

5. **Format: `pr`** — output a PR description:
```
## Summary

- Implements <intent>
- <scope-01>: <one-line scope description>
- <scope-02>: <one-line scope description>
…

## Scopes

| Scope | Status |
|-------|--------|
| scope-01-docs | ✅ DONE |
…

## Test Plan

- [ ] <done criterion from scope-01>
- [ ] <done criterion from scope-02>
…

## Planning reference
`.planning/active/<planning-id>/`
```

6. **Format: `tickets`** — output a numbered ticket list:
```
## Tickets — <planning-id>

1. **[scope-01-docs]** Document auth contract
   Area: DO
   Description: <scope objective>
   Acceptance criteria:
   - <done criterion 1>
   - <done criterion 2>

2. **[scope-02-api-domain]** …
```

7. Print the formatted output to the terminal. Do not write to any file unless the user pipes the output.

8. Add a footer note: `Generated by /plan-export from .planning/<state>/<planning-id>/` with the export date.

> Read-only. To save the output, pipe it: `/plan-export NNN-slug --format pr > pr-description.md`
