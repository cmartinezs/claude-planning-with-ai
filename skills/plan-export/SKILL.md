---
name: plan-export
description: Export a planning as a formatted document — PR description, ticket list, external issue draft, or standalone markdown summary.
argument-hint: <NNN-slug> [--format pr|tickets|github-issue|jira|linear|markdown]
allowed-tools: [Read, Bash, Glob]
---

Export a planning's content as a formatted document for use outside the planning system.

## Arguments

`$ARGUMENTS` — format: `NNN-slug [--format pr|tickets|github-issue|jira|linear|markdown]`
- `NNN-slug` — planning id (e.g. `001-jwt-auth-api`)
- `--format pr` — GitHub Pull Request description (default if in a git repo with a remote)
- `--format tickets` — numbered ticket list suitable for Linear or Jira
- `--format github-issue` — GitHub issue body per story, suitable for copy/paste or `gh issue create`
- `--format jira` — Jira issue draft per story
- `--format linear` — Linear issue draft per story
- `--format markdown` — standalone markdown document (default if no format specified)

## Steps

1. Parse `$ARGUMENTS`: extract planning id and optional `--format` flag. Default format: `markdown`.

2. Locate the planning in `.planning/` (INITIAL), `.planning/active/`, or `.planning/finished/`. If not found, stop and report.

3. Read `00-initial.md` (intent, why, scope), `01-expansion.md` (story list, dependencies, risk, external issue IDs), and all `02-deepening/*.md` story files (status, risk, done criteria, tasks).

4. **Format: `markdown`** — output a standalone document:
```
# <planning-id>: <intent>

**Why:** <why from 00-initial.md>

## Stories

| Story | Area | Status | Summary |
|-------|------|--------|---------|
| story-01-docs | DO | DONE | … |
…

## Done Criteria (per story)

### story-01-docs
- [ ] Auth contract documented in docs/api/auth.md
…

## Open Questions
<from 00-initial.md>
```

5. **Format: `pr`** — output a PR description:
```
## Summary

- Implements <intent>
- <story-01>: <one-line story description>
- <story-02>: <one-line story description>
…

## Stories

| Story | Status |
|-------|--------|
| story-01-docs | ✅ DONE |
…

## Test Plan

- [ ] <done criterion from story-01>
- [ ] <done criterion from story-02>
…

## Planning reference
`.planning/active/<planning-id>/`
```

6. **Format: `tickets`** — output a numbered ticket list:
```
## Tickets — <planning-id>

1. **[story-01-docs]** Document auth contract
   Area: DO
   Description: <story objective>
   Acceptance criteria:
   - <done criterion 1>
   - <done criterion 2>

2. **[story-02-api-domain]** …
```

7. **Format: `github-issue`, `jira`, or `linear`** — output one issue draft per story:
```
## Issue: [story-01-docs] Document auth contract

Planning: NNN-slug
Risk: M
Depends on: —
External ID: <existing external issue id, if present>

### Description
<story objective>

### Acceptance Criteria
- <done criterion 1>
- <done criterion 2>

### Implementation Notes
- Area: DO
- Expected outputs: <task output list>
```

8. Print the formatted output to the terminal. Do not write to any file unless the user pipes the output.

9. Add a footer note: `Generated by /plan-export from .planning/<state>/<planning-id>/` with the export date.

> Read-only. To save the output, pipe it: `/plan-export NNN-slug --format pr > pr-description.md`
