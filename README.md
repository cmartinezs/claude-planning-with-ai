## Install

From within Claude Code, add the marketplace and install the plugin:

```
/plugin marketplace add cmartinezs/claude-planning-with-ai
/plugin install claude-planning-with-ai@cmartinezs
```

Or from your terminal:

```bash
claude plugin marketplace add cmartinezs/claude-planning-with-ai
claude plugin install claude-planning-with-ai@cmartinezs
```

To update to the latest version later:

```bash
claude plugin update claude-planning-with-ai@cmartinezs
```

## First run

```bash
# Scan your project, propose area codes, scaffold .planning/
/plan-init
```

`/plan-init` detects your top-level directories, infers the stack (Next.js, Spring Boot, Terraform‚Ä¶), and proposes a short code per area (`WB`, `AP`, `IN`‚Ä¶). Confirm or adjust ‚ÄĒ then everything is wired up automatically.

## Work through the lifecycle

```bash
/plan-new 001-user-auth -- Implement JWT authentication
/plan-expand  001-user-auth           # INITIAL ‚Üí EXPANSION (identify stories)
/plan-atomize 001-user-auth story-01  # optional: atomic tasks (design + impl + unit tests)
/plan-task    001-user-auth story-01 task-01  # execute one atomic task
/plan-story   001-user-auth story-01  # DEEPENING (execute story tasks)
/plan-done    001-user-auth story-01  # mark story complete
/plan-archive 001-user-auth           # audit + move to finished/
```

Or generate a planning directly from existing stories:
```bash
/plan-from-epic 001 path/to/epic/   # 1 user story = 1 planning story
```

## Why this plugin

- **Lifecycle enforcement** ‚ÄĒ INITIAL ‚Üí EXPANSION ‚Üí DEEPENING ‚Üí COMPLETED. No skipping.
- **Area-aware traceability** ‚ÄĒ every story records which directories it touches (`AP`, `WB`, `IN`‚Ä¶).
- **Auto-configured for your project** ‚ÄĒ `/plan-init` discovers your structure; you confirm.
- **Backlog bridge** ‚ÄĒ convert existing epics/stories into executable plannings in one command.
- **Atomic task decomposition** ‚ÄĒ `/plan-atomize` turns a story into session-sized tasks, each with technical design, implementation steps, and unit tests.
- **Mid-execution adjustments** ‚ÄĒ enrich, split, or deepen stories without losing history. Switching context triggers git-state validation and safe alternatives (stash, WIP commit, STANDBY).
- **Self-checking** ‚ÄĒ `/plan-validate` audits structure, dependencies, and workflow references at any time.
- **Pure markdown** ‚ÄĒ no lock-in, readable in any editor, version-controlled with your code.
